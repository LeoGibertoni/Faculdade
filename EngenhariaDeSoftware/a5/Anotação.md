# Modelo Cascata



\- Abordagem sequencial p/ o desenvolvimento de software

\- Cada fase deve ser concluída antes que a próxima se inicie

\- As fases ocorrem de forma linear como uma cascata

\- Proposto por Wiston Royce em 1970

\- Fases principais



##### Levantamento de Requisitos

* Identificação e formalização dos requisitos



##### Projeto (Design)

* Definição de arquitetura



##### Implementação

* Escrita de código



##### Testes

* Valida se o software atende aos requisitos



##### Implantação

* Disponibilização de software



##### Manutenção

* Correção de problemas e melhoria



##### Gerenciamento de mudanças

* Modelo é rígido
* Alterações são custosas



##### Vantagens

* Simples
* Fácil de gerenciar



##### Desvantagens

* Rígido
* Descobertas de falhas tardias
* Feedback somente no fim



Adequado para projetos onde os requisitos são claros e bem compreendidos, e os requisitos não sofrem mudanças constantemente
